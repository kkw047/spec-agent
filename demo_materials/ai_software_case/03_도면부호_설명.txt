도면 설명:
도 1은 개발 문맥 기반 테스트 케이스 자동 제안 시스템의 전체 구성도입니다.
도 2는 변경 수집 모듈이 Git diff와 PR 설명을 수집하는 흐름을 나타내는 순서도입니다.
도 3은 코드 의존성 분석 모듈과 테스트 이력 벡터 저장소가 연결되는 데이터 흐름도입니다.
도 4는 테스트 제안 생성 모듈이 후보 테스트와 제안 이유를 생성한 후 사람 승인 인터페이스에 표시하는 화면 예시입니다.

부호의 설명:
100: 테스트 케이스 자동 제안 시스템
110: 변경 수집 모듈
120: 코드 의존성 분석 모듈
130: 테스트 이력 벡터 저장소
140: RAG 검색 모듈
150: 테스트 제안 생성 모듈
160: 사람 승인 인터페이스
170: 코드 저장소
180: CI 실행 서버
190: 개발자 단말

도면 관련 메모:
도면은 시스템 구성도와 데이터 흐름도 중심으로 작성합니다. UI 화면은 테스트 후보, 근거, 승인 상태가 보이도록 단순화합니다.
