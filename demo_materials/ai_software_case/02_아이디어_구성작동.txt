아이디어 메모

발명 명칭: 개발 문맥 기반 테스트 케이스 자동 제안 시스템

기술분야:
본 발명은 소프트웨어 코드 변경과 개발 문맥을 분석하여 테스트 케이스 후보를 제안하는 AI 기반 개발 보조 시스템에 관한 것입니다.

구성요소:
1. 변경 수집 모듈: Git diff, PR 설명, 이슈 본문, 커밋 메시지를 수집합니다.
2. 코드 의존성 분석 모듈: 변경 함수, 호출 관계, import 관계, 관련 설정 파일을 분석합니다.
3. 테스트 이력 벡터 저장소: 과거 테스트명, 결함 설명, 실패 로그, 수정 커밋 설명을 임베딩하여 저장합니다.
4. RAG 검색 모듈: 현재 변경과 유사한 과거 결함, 테스트, 코드 설명을 검색합니다.
5. 테스트 제안 생성 모듈: 단위 테스트, 통합 테스트, 회귀 테스트 후보와 이유를 생성합니다.
6. 사람 승인 인터페이스: 개발자가 테스트 후보를 채택, 수정, 제외하도록 합니다.

작동 방식:
개발자가 PR을 열거나 로컬 변경을 업로드하면 변경 수집 모듈이 변경된 파일, 함수, 이슈 설명을 추출합니다. 코드 의존성 분석 모듈은 변경 함수가 호출하는 모듈과 호출받는 모듈을 식별합니다. RAG 검색 모듈은 테스트 이력 벡터 저장소에서 유사한 결함 및 테스트 사례를 검색합니다. 테스트 제안 생성 모듈은 "왜 이 테스트가 필요한지"를 함께 포함한 테스트 후보 목록을 생성합니다. 사람 승인 인터페이스는 사용자가 후보를 확정하기 전에 수정할 수 있게 합니다.

실시예:
실시예 1은 웹 애플리케이션의 인증 로직 변경 사례입니다. 시스템은 인증 미들웨어 변경, 세션 만료 처리 변경, 관련 API 테스트 실패 이력을 검색하고 로그인, 토큰 갱신, 권한 거부 케이스를 테스트 후보로 제안합니다.

발명의 효과:
변경 파일만 보는 방식보다 간접 영향 테스트를 떠올리기 쉽습니다. 과거 장애나 실패 로그를 함께 참고하므로 회귀 테스트 후보를 설명 가능하게 제시할 수 있습니다. 테스트를 자동 확정하지 않고 사람이 승인하므로 책임 소재를 분리할 수 있습니다.
